# Laravel Eloquent 源代码分析

# 简介

在laravel 框架中，Eloquent ORM 的底层使用的是查询构造器实现的，Eloquent ORM中类封装了对数据表的 操作，使用时简单方便。 下面我们来简单分析下Eloquent ORM底层原理。

# 什么是 Eloquent

Eloquent 是 Laravel 的 'ORM'，即 'Object Relational Mapping'，对象关系映射。ORM 的出现是为了帮我们把对数据库的操作变得更加地方便。

# Eloquent ORM 创建

我们以 User::all() 的查询过程来作为本小节的开始

namespace App\Model

use Illuminate\Database\Eloquent\Model;

class User extends Model
{
    /**
     * 表名
     *
     * @var string
     */
    protected $table = 'user';


    /**
     * 表明模型是否应该被打上时间戳
     *
     * @var bool
     */
    public $timestamps = false;
}	

 $data = User::all();

在创建的模型类中其实什么都没有做，就可以实现对数据库操作相关功能 ，而这些功能都是继承了 Illuminate\Database\Eloquent\Model 类完成的。

# Eloquent ORM 底层实现

首先实例化自身，得到了模型，获取查询构造器，取得了模型数据，

/**
 * 从模型中获取所有数据集合
 *
 */
public static function all($columns = ['*'])
{
    return (new static)->newQuery()->get(
        is_array($columns) ? $columns : func_get_args()
    );
}

通过(new static)来到 Model 的构造方法，做哪些操作

  1. bootIfNotBooted标记模型被启动，并且触发模型启动的前置与后置事件
  2. syncOriginal保存原始对象数据
  3. fill初始化模型的属性
/**
* 创建Eloquent模型实例
* Create a new Eloquent model instance.
*
* @param  array  $attributes
* @return void
*/
public function __construct(array $attributes = [])
{
    // 1. 标记模型被启动，并且触发模型启动的前置与后置事件
    $this->bootIfNotBooted();
    
    // 2. 保存原始对象数据
    $this->syncOriginal();
    
    // 3. 初始化模型的属性
    $this->fill($attributes);
}

接下来将通过newQuery()方法生成Eloquent ORM查询构造器。 Eloquent ORM的底层使用了查询构造器来实现，这里通过newBaseQueryBuilder()方法创建一个基础查询构造器，这个基本构造器中的数据库连接最终通过数据库管理类Illuminate\Database\DatabaseManager的connection()方法实现的。

/**
* 获取一个针对连接的查询构造器
*
*/
protected function newBaseQueryBuilder()
{
    //  1.获取数据库连接
    $connection = $this->getConnection();

    // 2.实例化QueryBuilder 查询构造器
    return new QueryBuilder(
        $connection, $connection->getQueryGrammar(), $connection->getPostProcessor()
    );
}
